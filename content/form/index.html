Les components ExtJS
Le Form

Le form d'ExtJS permet de créer des formulaires facilement :
• Le layout du form est géré automatiquement grâce au "form layout" défini par défaut
• Toutes les options de configuration du BasicForm sont disponibles directement dans le FormPanel
• Les contrôles sur les champs sont facilités grâce à l'option vtype qui intègre les contrôles les plus courants : email, url, alpha, et alphanum.

Par défaut le formulaire est submit en Ajax, mais il est possible d'utiliser le submit classique avec l'option standardSubmit.



Les fields

Les champs du formulaires sont définis dans l'option items du FormPanel sous forme d'un tableau.

Chaque champ est défini avec un label, un name, et un xtype permettant de définir le type de champ (ex : checkbox, combo, datefield, displayfield, radio, textarea, textfield, timefield...). Toutes les options de configuration du xtype en question sont ensuite disponibles.

Si le xtype n'est pas défini alors le type de champ par défaut sera utilisé. C'est l'option defaultType du FormPanel qui permet de définir le type de champ par défaut (ex ici: textfield).

items: [
	{fieldLabel: 'First Name', name: 'first', allowBlank: false},
	{fieldLabel: 'Last Name', name: 'last'},
	{fieldLabel: 'Email', name: 'email', vtype: 'email'}
]


Les boutons

L'option buttons du FormPanel permet d'ajouter des bouttons dans le footer du FormPanel.

L'option type permet de définir le type de boutton : 'submit', 'reset', ou 'button'.
Par défaut le type est 'button'. On utilise alors un handler pour définir le comportement à appliquer au boutton.

buttons:[{
	text: 'Save',
	handler: function() {
		saveForm();
	}
}, {
	text: 'Cancel',
	handler: function() {
		cancelForm();
	}
}]


Les valeurs par défaut

L'option defaults du FormPanel permet de définir des valeurs par défaut qui seront appliquées à tous les items du FormPanel. Elle récupère une liste d'options de configuration applicable à chaque item.

ex : forcer tous les champs à être remplis en rajoutant dans defaults "allowBlank: false", appliquer un layout à tous les champs, ...

defaults: {anchor:'0'}


La validation

ExtJS fournit une classe (singleton) Vtypes contenant les fonctions de validation basiques les plus couramment utilisées dans les formulaires (email, url, caractères spéciaux, ...).
L'ajout de l'option vtype à la définition d'un champ intègre le contrôle voulu.

Il est possible de créer ses propres vtype (ex: time, IP,...).

{fieldLabel: 'Email', name: 'email', vtype: 'email'}
