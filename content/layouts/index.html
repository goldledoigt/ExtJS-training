Les Layouts ExtJS

Le layout engine de ExtJS permet de prendre en charge les problématiques de positionnement cross-browser.
Il existe différents layouts permettant d'obtenir différents modes de positionnement des élements dans un container.

  • Exemples


Viewport

Le Viewport est le container qui représente la fenêtre du browser.
Il est render dans le document.body et s'adapte automatiquement aux changements de tailles du browser.
C'est lui qui va contenir l'ensemble des élements de votre application ExtJS. On l'utilise généralement avec un BorderLayout.


Border Layout

Le Border Layout est le type de layout le plus couramment utilisé.
Il décompose l'espace en régions : nord, sud, est, ouest, et centre.
Seule la région centre est obligatoire.

  • Exemple de border layout


Colomn Layout

Le layout Column est utile pour disposer des élements de hauteur variable dans plusieurs colonnes.
Table Layout

Le layout Table permet de gérer la disposition des élements comme dans un tableau HTML.
Accordion Layout

Le layout Accordion permet de créer une pile de panels, 1 seul panel pouvant être affiché à la fois.
Vbox & Hbox Layout


Le layout Vbox permet de disposer les éléments en lignes, avec la possiblité de les étirer verticalement afin de remplir parfaitement le container parent.



Le layout Hbox permet de disposer les éléments en colonnes, avec la possibilité de les étirer horizontalement afin de remplir le container parent.

Nested layouts

Chaque container peut avoir son propre layout. Ainsi on peut combiner différents types de layouts afin de créer toutes sortes d'interfaces.
Les possibilités sont infinies.
